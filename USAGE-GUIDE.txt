================================================================================================
                           API Automation Agent - Usage Guide
================================================================================================

PREREQUISITES
- Windows 7+ or macOS 10.14+
- Node.js (download from https://nodejs.org/)
- API key (OpenAI, Anthropic, Google, or AWS Bedrock credentials)
- Internet connection

================================================================================================
SETUP STEPS
================================================================================================

INTERACTIVE SETUP (RECOMMENDED - FIRST TIME USE):
When you run the CLI for the first time, it will automatically guide you through setup:
1. Select AI Provider (Anthropic, OpenAI, Google, or AWS Bedrock)
2. Choose Model from supported options  
3. Enter your API key or AWS credentials
The CLI will create a .env file automatically.

SUPPORTED MODELS:

- claude-sonnet-4-5-20250929 (default - best balance)
- claude-haiku-4-5-20251001 (fast + low cost)

- claude-sonnet-4-20250514

OpenAI Models:
- gpt-5.1 (recommended)
- gpt-5

Google Models:

AWS Bedrock Models:

- anthropic.claude-sonnet-4-20250514-v1:0

- google.gemini-3-pro-preview

MANUAL SETUP (ALTERNATIVE):

WINDOWS:
1. Extract api-agent-windows.zip
2. Open Command Prompt or PowerShell in the extracted folder
3. Copy example.env to .env:
   copy example.env .env
4. Edit .env in Notepad and add your API key:
   OPENAI_API_KEY=your_key_here
   OR
   ANTHROPIC_API_KEY=your_key_here
   OR
   GOOGLE_API_KEY=your_key_here
   OR (for AWS Bedrock):
   AWS_ACCESS_KEY_ID=your_access_key_id
   AWS_SECRET_ACCESS_KEY=your_secret_key
   AWS_REGION=us-east-1

MACOS:
1. Extract the tar.gz file:
   tar -xzf api-agent-macos.tar.gz
   cd api-agent-macos
2. Copy example.env to .env:
   cp example.env .env
3. Edit .env with any text editor and add your API key:
   OPENAI_API_KEY=your_key_here
   OR
   ANTHROPIC_API_KEY=your_key_here
   OR
   GOOGLE_API_KEY=your_key_here
   OR (for AWS Bedrock):
   AWS_ACCESS_KEY_ID=your_access_key_id
   AWS_SECRET_ACCESS_KEY=your_secret_key
   AWS_REGION=us-east-1
4. Make executable (if needed):
   chmod +x api-agent

================================================================================================
BASIC USAGE
================================================================================================

WINDOWS:
- Show help: ./api-agent --help
- Generate framework: ./api-agent swagger.json
- Custom folder: ./api-agent swagger.json --destination-folder my-tests

MACOS:
- Show help: ./api-agent --help
- Generate framework: ./api-agent swagger.json
- Custom folder: ./api-agent swagger.json --destination-folder my-tests

================================================================================================
ADVANCED USAGE
================================================================================================

List available endpoints:
    ./api-agent --list-endpoints

Generate framework for specific endpoints:
    ./api-agent --destination-folder my-framework --endpoints /pets /users

Generate with different options:
    Models only:
    ./api-agent --generate models --destination-folder my-framework

    Models and first test:
    ./api-agent --generate models_and_first_test --destination-folder my-framework

    Models and all tests:
    ./api-agent --generate models_and_tests --destination-folder my-framework

================================================================================================
EXAMPLE COMPLETE COMMAND
================================================================================================

./api-agent --destination-folder my-pet-store-tests --endpoints /pets /store --generate models_and_tests

================================================================================================
TROUBLESHOOTING
================================================================================================

1. "Missing API key" error:
   - Make sure you have a ".env" file in the same folder as the executable
   - Check that your API key is correctly formatted in the ".env" file

2. "Internet connection" error:
   - The tool requires internet to communicate with AI APIs
   - Check your internet connection

3. "Permission denied" error:
   - Make sure you're running the command prompt as an administrator
   - Or try running from a folder where you have write permissions

4. Node.js not found:
   - Install Node.js from https://nodejs.org/
   - Restart your terminal
   - Verify: node --version

5. Permission denied (macOS):
   - Run: chmod +x api-agent

6. API key issues:
   - Check .env file has correct key without extra spaces
   - Ensure API key has sufficient permissions/credits
